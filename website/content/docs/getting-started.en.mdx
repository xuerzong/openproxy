---
title: Getting Started
description: Start OpenProxy locally in about 10 minutes
---

This guide helps you launch the core `OpenProxy` services quickly.

## Prerequisites

- Bun `>= 1.3.5`
- Rust (latest stable recommended)
- PostgreSQL (local or remote)

## 1) Install dependencies

Run in the repository root:

```bash
bun install
```

## 2) Configure environment variables (Server)

```bash
cp apps/server/.env.example apps/server/.env
```

At minimum, configure these variables:

- `DATABASE_URL`
- `BETTER_AUTH_SECRET`
- `BETTER_AUTH_TRUSTED_ORIGINS`
- `BETTER_AUTH_URL`
- `RSA_PRIVATE_KEY`
- `RSA_PUBLIC_KEY`

## 3) Generate an RSA key pair

Run in the repository root:

```bash
bun scripts/generateRSAKey.ts
```

Copy the output values into `apps/server/.env` for `RSA_PRIVATE_KEY` and `RSA_PUBLIC_KEY`.

## 4) Run database migrations

```bash
cd apps/server
bun run migrate
```

## 5) Start services

Run in the repository root (recommended):

```bash
bun run dev
```

This starts all workspaces that provide a `dev` script through Turborepo.

## 6) URLs

- Server default: `http://localhost:3888`
- Rust API Proxy default: `http://localhost:5060`
- Web dev (tenant/admin): Vite URL (usually `http://localhost:5173`)

## Common startup issues

### Database connection failed

Check whether `DATABASE_URL` is correct and PostgreSQL is running with proper permissions.

### Auth callback issues

Check whether `BETTER_AUTH_URL` and `BETTER_AUTH_TRUSTED_ORIGINS` match your current development origin.

### API key decryption failed

Ensure key pair values are complete and not broken by invalid line breaks or spaces. `RSA_PRIVATE_KEY` and `RSA_PUBLIC_KEY` must come from the same generation.
