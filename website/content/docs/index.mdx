`OpenProxy` 是一套可自托管的 AI API 代理网关。它部署在你的应用与上游 AI 服务商（OpenAI、Anthropic、阿里百炼等）之间，对外提供统一的 OpenAI/Anthropic 兼容接口。鉴权、配额校验、用量统计与多供应商路由全部在网关内完成，应用层只需对接一个地址。

## 核心特性

- **统一 API**：兼容 OpenAI（`/v1/chat/completions`、`/v1/embeddings`）和 Anthropic（`/v1/messages`）接口
- **多供应商容灾**：同一模型可绑定多个 AI 后端，按权重随机选取，失败时自动依次回退
- **多 Key 轮换**：每个供应商可配置多个 API Key。代理追踪每个用户 API Key 最近 N 次命中的 `(供应商, Key)` 组合（N = min(10, 组合总数)），优先选取最近未使用的组合，降低触发限速风险
- **API Key 管理**：支持额度、请求次数、过期时间、模型访问控制
- **用量统计**：按 token 计费、记录响应时间与供应商归属
- **密钥加密存储**：供应商 API Key 使用 RSA 加密存储，运行时解密并缓存（内存，1 小时 TTL）
- **多方式认证**：邮箱、魔法链接、手机 OTP、GitHub、Google（集成 better-auth）
- **管理后台**：模型、供应商、用户、API Key、订单管理
- **租户仪表盘**：用量图表、余额展示、历史请求

## 项目组成

| 组件 | 技术栈 | 职责 |
| --- | --- | --- |
| `apps/api` | Rust + Axum | 代理核心：路由、鉴权、用量回写 |
| `apps/server` | Bun + Elysia | 业务后端：用户、密钥、模型、计费 |
| `apps/web` | React + Vite | 租户仪表盘 + 管理后台 |
| `packages/schema` | Zod / TypeBox | 共享校验 Schema |
| `packages/payment-provider` | TypeScript | 支付接口抽象 |
| `packages/phone-auth` | TypeScript | 手机 OTP 鉴权 |
| `website` | Next.js + Fumadocs | 本文档站 |
