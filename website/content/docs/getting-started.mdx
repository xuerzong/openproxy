---
title: 快速开始
description: 10 分钟完成 OpenProxy 本地启动
---

本文档用于快速启动 `OpenProxy` 的核心服务。

## 前置条件

- Bun `>= 1.3.5`
- Rust（建议使用最新版 stable）
- PostgreSQL（本地或远程均可）

## 1) 安装依赖

在仓库根目录执行：

```bash
bun install
```

## 2) 配置环境变量（Server）

```bash
cp apps/server/.env.example apps/server/.env
```

最少需要配置这些变量：

- `DATABASE_URL`
- `BETTER_AUTH_SECRET`
- `BETTER_AUTH_TRUSTED_ORIGINS`
- `BETTER_AUTH_URL`
- `RSA_PRIVATE_KEY`
- `RSA_PUBLIC_KEY`

## 3) 生成 RSA 密钥对

在仓库根目录执行：

```bash
bun scripts/generateRSAKey.ts
```

将输出填入 `apps/server/.env` 的 `RSA_PRIVATE_KEY` 和 `RSA_PUBLIC_KEY`。

## 4) 执行数据库迁移

```bash
cd apps/server
bun run migrate
```

## 5) 启动服务

在仓库根目录执行（推荐）：

```bash
bun run dev
```

这会通过 Turborepo 启动所有有 `dev` 脚本的子应用。

## 6) 访问地址

- Server 默认监听：`http://localhost:3888`
- Rust API Proxy 默认监听：`http://localhost:5060`
- Web 本地开发（tenant/admin）默认使用 Vite 地址（通常 `http://localhost:5173`）

## 常见启动问题

### 数据库连接失败

检查 `DATABASE_URL` 是否正确，确认 PostgreSQL 已启动且账号有权限。

### 鉴权回调异常

检查 `BETTER_AUTH_URL` 与 `BETTER_AUTH_TRUSTED_ORIGINS` 是否和当前开发域名一致。

### API Key 解密失败

确保密钥对完整且未破坏换行/空格格式，`RSA_PRIVATE_KEY` 和 `RSA_PUBLIC_KEY` 必须来自同一次生成。
