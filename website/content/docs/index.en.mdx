`OpenProxy` is a self-hosted AI API gateway. It sits in front of multiple AI provider backends (OpenAI, Anthropic, Alibaba Bailian, etc.) and exposes a unified OpenAI/Anthropic-compatible interface to your application. All authentication, quota enforcement, usage tracking, and provider routing happen inside the gateway — your application only ever talks to one endpoint.

## Key Features

- **Unified API** — OpenAI-compatible (`/v1/chat/completions`, `/v1/embeddings`) and Anthropic-compatible (`/v1/messages`) endpoints
- **Multi-provider failover** — Assign multiple AI backends per model. The proxy selects one via weighted random, then falls through the remaining combos automatically on failure
- **Multi-key rotation** — Each provider can have multiple API keys. The proxy tracks the last N used `(provider, key)` pairs per user API key (N = min(10, total pairs)) and prefers untried pairs first, reducing rate-limit risk
- **API key management** — Per-key quota, request limits, expiry, and model access control
- **Usage tracking** — Token-level cost accounting, response time, and provider attribution per request
- **Secure key storage** — Provider API keys are RSA-encrypted at rest; decrypted in-process with a 1-hour in-memory cache
- **Authentication** — Email/password, magic link, phone OTP, GitHub, Google (via [better-auth](https://better-auth.com))
- **Admin dashboard** — Manage models, providers, users, API keys, and orders
- **Tenant dashboard** — Usage charts, balance, and request history

## Project Components

| Component | Technology | Role |
| --- | --- | --- |
| `apps/api` | Rust + Axum | Proxy core: routing, key auth, usage write-back |
| `apps/server` | Bun + Elysia | Business backend: users, keys, models, billing |
| `apps/web` | React + Vite | Tenant dashboard + admin panel |
| `packages/schema` | Zod / TypeBox | Shared validation schemas |
| `packages/payment-provider` | TypeScript | Payment integration abstraction |
| `packages/phone-auth` | TypeScript | Phone OTP auth |
| `website` | Next.js + Fumadocs | This documentation site |
